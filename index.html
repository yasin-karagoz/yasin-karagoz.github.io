About

Staff Software Engineer with a career spanning Systems Administration, Infrastructure Engineering, DevOps, and Staff-level Engineering. I specialize in finding gaps in existing IaC and codebases and closing them with modern, production-grade automation — so teams ship faster and infrastructure becomes a competitive advantage, not a bottleneck.

  • I love finding gaps in infrastructure and closing them. My go-to stack is GitHub Actions, Terraform, Helm, ArgoCD, and GKE, and I'm always looking for ways to turn repetitive work into something a team can trigger with a single PR.
  • I write custom Golang Kubernetes operators and controllers when off-the-shelf tools aren't enough. If it can be automated, I'll build the thing that automates it.
  • I've led FinOps projects on both AWS and GCP, building custom operators that surface resource waste before it becomes a problem. Cloud bills have a way of surprising people, and I like making sure they don't.
  • One of my favorite things I've built is a self-service database provisioning system. Developers open a PR to request MySQL, PostgreSQL, AlloyDB, or MongoDB, and a production-ready instance is waiting for them when it merges.
  • At the end of the day, I care about making infrastructure invisible to the teams using it. If a developer has to think too hard about the platform, that's a problem worth solving.
